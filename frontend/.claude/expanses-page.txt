================================================================
ZELO — MÓDULO DE DESPESAS v2.0
================================================================

OBJETIVO
Controlar contas a pagar do estabelecimento, separando despesas
pontuais de recorrentes mensais, com geração automática via jobs.

================================================================
BANCO DE DADOS
================================================================

recurring_expense_templates
  id              CHAR(36) PK
  tenant_id       VARCHAR(36) NOT NULL
  description     VARCHAR(255) NOT NULL
  category        ENUM(Aluguel, Fornecedor, Energia, Água, Internet,
                       Salários, Marketing, Impostos, Manutenção, Outros)
  value           DECIMAL(10,2) NOT NULL
  due_day         TINYINT NOT NULL  -- dia do mês (1–28)
  next_renew_date DATE NOT NULL     -- próxima data que o job vai gerar
  is_active       BOOLEAN DEFAULT TRUE
  created_at / updated_at

expenses
  id              CHAR(36) PK
  tenant_id       VARCHAR(36) NOT NULL
  template_id     CHAR(36) NULL  -- FK para templates (NULL se pontual)
  description     VARCHAR(255) NOT NULL
  category        ENUM(mesmos valores)
  value           DECIMAL(10,2) NOT NULL
  due_date        DATE NOT NULL
  status          ENUM(pendente, pago) DEFAULT 'pendente'
  is_recurring    BOOLEAN DEFAULT FALSE
  created_at / updated_at

OBSERVAÇÃO: nextRenew não existe em expenses. Pertence ao template.
A UI lê template.next_renew_date quando is_recurring = true.

================================================================
JOBS — HANGFIRE (ASP.NET + MySQL)
================================================================

CONFIGURAÇÃO (Program.cs)
  builder.Services.AddHangfire(config =>
      config.UseStorage(new MySqlStorage(connectionString)));
  builder.Services.AddHangfireServer();

  RecurringJob.AddOrUpdate<ExpenseRenewalJob>(
      "expense-renewal", job => job.RunAsync(), Cron.Daily(0, 0));

  RecurringJob.AddOrUpdate<ExpenseDueSoonJob>(
      "expense-due-soon", job => job.RunAsync(), Cron.Daily(8, 0));

JOB 1 — ExpenseRenewalJob (meia-noite)
  - Busca templates onde is_active = true AND next_renew_date <= hoje
  - Para cada um:
      · Cria ocorrência em expenses (status = pendente)
      · Avança template.next_renew_date += 1 mês

JOB 2 — ExpenseDueSoonJob (8h)
  - Busca expenses onde status = pendente AND due_date = amanhã
  - Para cada uma:
      · Insere registro em notifications (type = "bill_due")
  - Desacoplado: só grava na tabela. O módulo de notificações
    consome depois.

================================================================
REGRAS DE RECORRÊNCIA
================================================================

CRIAR
  - recurring = true → cria template + primeira ocorrência em expenses
  - due_day = dia do campo due; next_renew_date = due + 1 mês

EDITAR
  - Perguntar escopo:
      "Somente esta" → atualiza só expenses
      "Esta e futuras" → atualiza template + esta ocorrência

EXCLUIR
  - Perguntar escopo:
      "Somente esta" → remove ocorrência de expenses
      "Parar recorrência" → is_active = false no template + remove ocorrência

SUGESTÃO DE nextRenew NO MODAL
  - Ao preencher due, se recurring = true e renewTouched = false:
    sugerir nextRenew = due + 1 mês automaticamente
  - Se usuário editar nextRenew manualmente: renewTouched = true


================================================================
ENTIDADE Expense (DTO de resposta)
================================================================

  id, templateId (null se pontual), desc, category,
  value, dueDate (ISO), status, isRecurring,
  nextRenewDate (lido do template, null se pontual)

================================================================
KPIs (calculados do mês de referência)
================================================================

  Total do mês   → soma de value
  Pagas          → soma onde status = pago  
  A pagar        → soma onde status = pendente 
  Recorrentes    → soma onde is_recurring = true

  Por categoria  → { [cat]: soma } — donut com CAT_COLORS, legenda desc por valor
  Impacto anual  → value * 12 (exibido no modal ao ativar recurring)

================================================================
LISTAGEM
================================================================

Colunas: Descrição (tag "Mensal" se recorrente) | Categoria (dot colorido)
         | Vencimento DD/MM | Renovação DD/MM ou "—" | Status | Valor | Ações

Filtros: Todas (default) | Recorrentes | Pontuais

Ações por linha: Editar | Marcar paga/pendente | Duplicar | Excluir

================================================================
MODAL NOVA DESPESA
================================================================

Estado inicial:
  desc="", cat="", val="", due="2026-05-22", status="pendente",
  recurring=false, nextRenew="2026-06-22", renewTouched=false

Campos:
  (a) Descrição — texto obrigatório
  (b) Valor     — decimal pt-BR, adornment "R$", parser remove "." e troca "," por "."
  (c) Vencimento — date, dispara sugestão de nextRenew se !renewTouched
  (d) Categoria  — chip-picker 10 opções, obrigatório
  (e) Status     — chip: "A pagar" (pendente) | "Já paga" (pago)
  (f) Recorrente — toggle; quando ON revela nextRenew + card impacto anual

Fechar: Esc | clique fora | botão ×
Toast de confirmação: 3 segundos após salvar

================================================================
REGRAS DE NEGÓCIO
================================================================

R1.  value > 0, desc não vazia, category e due_date obrigatórios
R2.  Status default: pendente. Transição via PATCH /status ou modal
R3.  recurring=true → template obrigatório com next_renew_date
R4.  Pontual → template_id = null
R5.  ExpenseRenewalJob roda à meia-noite; gera ocorrência e avança template
R6.  ExpenseDueSoonJob roda às 8h; gera notificação bill_due para vencimentos amanhã
R7.  Editar recorrente: escopo single altera expenses; future altera template
R8.  Excluir recorrente: escopo single remove ocorrência; stop desativa template
R9.  Aggregações consideram ocorrências do mês de referência
R10. Donut usa CAT_COLORS fixas
R11. Toast 3s após salvar
R12. Campos obrigatórios com asterisco vermelho + erro inline
